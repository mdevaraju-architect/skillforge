# 01 — Architecture: Salesforce Collaborative Forecasting Object Model

## Object Relationship Overview

```
ForecastingType
│
│  (one ForecastingType generates one tree of ForecastingItem records)
│
└─► ForecastingItem  [READ-ONLY — platform-generated]
      │
      │  (one ForecastingItem per user × period × ForecastCategory × ForecastingType)
      │
      ├─► ForecastingOwnerAdjustment   (owner adjusts their own row)
      ├─► ForecastingManagerAdjustment (manager adjusts a direct report's row)
      └─► ForecastingSubmission        (point-in-time snapshot of a manager's forecast)

ForecastingQuota
      (matched to ForecastingItem by StartDate + ForecastingTypeId + QuotaOwnerId;
       no lookup field — join in SOQL/Apex)

OpportunitySplit
      (enables overlay forecasting; SplitType = Overlay for overlay reps)
      (SplitType = Revenue for revenue credit splits summing to 100%)

Opportunity  ──────────────────────────────────────────────────────────►  ForecastingItem
  .CloseDate                                                               .StartDate  (period bucket)
  .ForecastCategoryName  (Pipeline / BestCase / Commit / Closed / Omitted) .ForecastCategoryName
  .Amount  (or custom field per ForecastingType)                           .OwnerOnlyAmount
  .OwnerId                                                                 .OwnerId
```

## `ForecastingItem` Key Fields

`ForecastingItem` records are auto-generated by the platform. They are read-only via DML.

| Field API Name | Type | Description |
|---|---|---|
| `Id` | ID | Standard record ID |
| `OwnerId` | Lookup(User) | The user this row belongs to in the hierarchy |
| `ForecastingTypeId` | Lookup(ForecastingType) | Which forecast measure this row belongs to |
| `StartDate` | Date | First day of the forecast period (e.g. `2026-07-01` for Q3 or July) |
| `EndDate` | Date | Last day of the forecast period |
| `PeriodType` | Picklist | `Month` or `Quarter` — matches ForecastingType period config |
| `ForecastCategoryName` | Picklist | `Pipeline`, `BestCase`, `Commit`, `Closed`, `Omitted` (or custom) |
| `ForecastingItemCategory` | Picklist | Internal category code used by the platform |
| `ForecastAmount` | Currency | Full rollup including all subordinates and all adjustments — **do not sum across hierarchy** |
| `OwnerOnlyAmount` | Currency | Only this user's direct opportunities, no subordinate rollup — use for rep-level analysis |
| `AdjustedAmount` | Currency | Manager-adjusted amount layered on top of the rollup |
| `OwnerAdjustedAmount` | Currency | Owner's own adjustment to their row |
| `IsUpdatable` | Boolean | Whether adjustments are currently permitted on this row |
| `HasAdjustment` | Boolean | True if any adjustment exists on this row |
| `HasOwnerAdjustment` | Boolean | True if an owner adjustment exists |
| `CurrencyIsoCode` | Picklist | Active in multi-currency orgs |

### ForecastAmount Aggregation Warning

```
        EVP (ForecastAmount = $5M)
        └── VP (ForecastAmount = $5M)
             ├── Manager A (ForecastAmount = $3M)
             │    ├── Rep 1 (OwnerOnlyAmount = $1.5M)
             │    └── Rep 2 (OwnerOnlyAmount = $1.5M)
             └── Manager B (ForecastAmount = $2M)
                  ├── Rep 3 (OwnerOnlyAmount = $1M)
                  └── Rep 4 (OwnerOnlyAmount = $1M)

SUM(ForecastAmount) for all rows = $5M + $5M + $3M + $1.5M + $1.5M + $2M + $1M + $1M = $20M  ← WRONG (8x overcounting)
SUM(OwnerOnlyAmount) for leaf reps only = $1.5M + $1.5M + $1M + $1M = $5M  ← CORRECT
```

## `ForecastingType` Key Fields

`ForecastingType` is configured via Setup → Forecast Settings. It is queryable via SOQL.

| Field API Name | Type | Description |
|---|---|---|
| `Id` | ID | Standard record ID |
| `DeveloperName` | String | API name (e.g. `OpportunityRevenue`) |
| `MasterLabel` | String | Display name in UI |
| `IsActive` | Boolean | Whether this type is currently active and generating ForecastingItem records |
| `ForecastingHierarchyType` | Picklist | `Role` (role-based) or `RoleTerritory2` (territory-based) |
| `ForecastedAmountFieldApiName` | String | The Opportunity field that rolls up (e.g. `Amount`, `CustomRevenue__c`) |
| `PeriodType` | Picklist | `Month` or `Quarter` — changing this value is destructive |
| `RollupQuerySourceColumn` | String | Internal — identifies the source measure (Revenue, Quantity, etc.) |
| `HasProductFamily` | Boolean | True if this is a product family forecast type |
| `DisplayedCategoryApiNames` | String | Comma-delimited list of forecast categories exposed in this type |

## `ForecastingOwnerAdjustment` Key Fields

| Field API Name | Type | Description |
|---|---|---|
| `Id` | ID | Standard record ID |
| `ForecastingItemId` | Lookup(ForecastingItem) | The ForecastingItem row being adjusted |
| `OwnerId` | Lookup(User) | The user making the adjustment (must be the ForecastingItem owner) |
| `AdjustedAmount` | Currency | The adjusted value |
| `OldAmount` | Currency | The original unadjusted value at time of adjustment |
| `IsAmount` | Boolean | True = absolute override; False = delta adjustment |
| `ForecastCategoryName` | Picklist | Which category column is being adjusted |
| `PeriodId` | Lookup | The period this adjustment covers |
| `ForecastingTypeId` | Lookup(ForecastingType) | Which forecast type this adjustment applies to |

| Field API Name | Type | Description |
|---|---|---|

## `ForecastingQuota` Key Fields

| Field API Name | Type | Description |
|---|---|---|
| `Id` | ID | Standard record ID |
| `QuotaOwnerId` | Lookup(User) | The user the quota is assigned to |
| `StartDate` | Date | First day of the quota period — must match ForecastingItem StartDate exactly |
| `QuotaAmount` | Currency | The quota value |
| `ForecastingTypeId` | Lookup(ForecastingType) | Which forecast type this quota applies to |
| `CurrencyIsoCode` | Picklist | Active in multi-currency orgs |
| `ProductFamily` | String | For product family forecast types, the product family bucket |

**Join key for attainment:** `ForecastingQuota.QuotaOwnerId` = `ForecastingItem.OwnerId` AND `ForecastingQuota.StartDate` = `ForecastingItem.StartDate` AND `ForecastingQuota.ForecastingTypeId` = `ForecastingItem.ForecastingTypeId`.

---

## `OpportunitySplit` Key Fields

| Field API Name | Type | Description |
|---|---|---|
| `Id` | ID | Standard record ID |
| `OpportunityId` | Lookup(Opportunity) | The opportunity this split belongs to |
| `SplitOwnerId` | Lookup(User) | The user receiving the split credit |
| `SplitPercentage` | Percent | Credit percentage — revenue splits must total 100%; overlay splits are additive |
| `SplitTypeId` | Lookup(OpportunitySplitType) | References the split type (Revenue, Overlay, or custom) |
| `SplitAmount` | Currency | Calculated: `Opportunity.Amount × SplitPercentage / 100` |

## Object Dependency Summary

```
Opportunity (source data)
    ↓  (platform rolls up on CloseDate + ForecastCategoryName + OwnerId)
ForecastingItem  (read-only, one per user × period × category × type)
    ↓
ForecastingOwnerAdjustment  (owner adjusts their own row — DML allowed)
ForecastingManagerAdjustment  (manager adjusts direct report's row — DML allowed)
ForecastingSubmission  (point-in-time snapshot — insert creates snapshot, no update)

ForecastingQuota  (independent object, no FK to ForecastingItem — join by matching fields)

OpportunitySplit  (source data for overlay/revenue split forecast types)
    ↓  (platform rolls up SplitAmount for users with overlay ForecastingType)
ForecastingItem  (overlay row for the split recipient)
```
